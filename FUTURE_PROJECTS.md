## 1. Voice Input / Dictation Project

### Status

- parked
- not an active build priority
- keep as future vibe-coding candidate

### Why It Is Parked

The market is crowded.

Practical reason:

- voice input and AI dictation are currently being built by many teams and solo builders
- generic dictation is increasingly commoditized
- it is hard to win without strong UX, distribution, a narrow niche, or a very specific workflow advantage

This means:

- the research is still useful
- the category is still interesting
- but it is not currently the best use of build effort

### What Has Already Been Done

This repository already contains:

- category separation between dictation, managed APIs, and self-hosted ASR
- personal dictation tool research
- managed STT API research
- open source/self-host survey
- implementation blueprint for a backend service

So if the project becomes active later, the restart cost is low.

### Conditions For Reopening

Reopen this project only if at least one of these is true:

- a clear niche appears
- a strong distribution channel appears
- a specific workflow is identified where current tools are weak
- a private/offline/Russian/developer-first angle becomes compelling
- a customer asks for it directly

### Better Angles Than Generic Dictation

If this topic is revisited, do not reopen it as:

- "another general AI voice input tool"

Prefer a narrower angle such as:

- Russian-first dictation
- developer-first dictation
- privacy-first offline dictation
- dictation + coding actions
- dictation for a specific professional workflow
- dictation + transcript cleanup for meetings or interviews
